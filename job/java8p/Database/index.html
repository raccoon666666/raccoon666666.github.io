01.yml Q: 索引的类型有哪些?

A: 
  物理类型:
    B+树:  
      - 遍历到叶子获得Key,不一定回表查询
    Hash: 
      - 先Hash获得Key,然后回表查询
      - 等值查询快,但重复值多时容易碰撞性能下降
      - 无法范围查询(Hash丢失了顺序)
      - 不能用索引排序、联合索引、模糊查询
  使用类型:
    主键索引: Primary Key,特殊的唯一索引,不允许重复,不允许NULL
    唯一索引: Unique,不允许重复,允许NULL
    普通索引: 允许重复,允许NULL
    组合索引: 列值的组合必须唯一
    全文索引: Fulltext,对(CHAR、VARCHAR、TEXT)建立的所有,支持值的全文查找
    空间索引: Spatial,对空间数据类型(非NULL)(GEOMETRY、POINT、LINESTRING、POLYGON)建立的索引

02.yml Q: 哪些字段要使用索引,什么时候生效? 哪些不要用,什么时候失效?

A: 
  适用:
    - 常查询 => 普通索引 => 加速查询
    - 外键字段 => 普通索引 => 加速表连接
    - 排序字段 => 组合索引 => 加速组合排序
    - 唯一性的列 => 唯一索引 => 保证数据完整性
  不适用:
    - 频繁更新的字段
    - 数据量较少的表
    - 数据不同值较少的列(男女)
  失效:  # explain语句查看查询流程
    - 字段参与了计算、函数、隐式转换、显式转换
    - is NULL、is NOT NULL、!=
    - 通配符在最前的模糊查询(like '%111')(可使用reverse改进)
    - 组合索引中没有按照左前缀匹配(无头,中断)
    - 组合索引中某字段范围比较,若之后不是范围比较则失效
    - 使用or连接条件

03.yml Q: 索引的底层存储原理是什么?

A: 
  MyISAM:
    - 使用B+树,索引与数据分离
    - 主索引和辅助索引相同,记录了
  InnoDB:
    存储:
      - 使用B+树,索引与数据合一
      - 主索引(聚簇索引),记录了
      - 辅助索引,记录了  
      - 联合索引,记录了
    要求:
      - 要求表必须有主键,否则自动选择某列,再否则自动生成long6字段作主键
      - 辅助索引搜索时要搜索两次
      - Pkey不宜过长,因为每个辅助索引存储的是Pkey本身
      - Pkey要自增,否则构造B+树时要频繁分裂插值

04.yml Q: 索引的作用有哪些?弊端有哪些?

A: 
  作用:
    - 加速表查询、连接、分组、排序
    - 使用唯一索引保证行的唯一性
  弊端:
    - 创建索引需要时间
    - 索引文件占磁盘空间
    - 只加速了查,增删改时需同步更新索引

05.yml Q: 什么时候会索引重建?如何重建?重建的原理是什么?

A: 
  时机:
    - 表上频繁发生增删改
    - 对表move使行的id发生变化
    - height>=4或DEL_LF_ROWS/LF_ROWS>0.2
  方法:
    - 先drop然后create,耗时长
    - alter rebuild,耗时短
  原理:
    - 在旧索引基础上重建,并写入备份文件,防止意外丢失
    - 使用online参数减少加锁问题;生成系统临时日志表(记录索引变化)
    -                  获取数据      重建时排序   阻塞DML
    - rebuild         扫描旧索引        有         会
    - rebuild online    扫描表          有        不会      

06.yml Q: 说说你对数据库数据完整性的理解

A: 
  作用: 使数据库能真实反应现实世界
  功能:
    - 允许用户定义数据完整性约束条件
    - 对于提交的请求检查是否违背完整性(分为立即检查、延迟检查)
    - 对于违背完整性的操作要采取动作
  实体完整性: 主键不能重复,不能为NULL
  参照完整性: 不能为NULL
  用户定义完整性: 满足表达式

07.yml Q: 为什么选择B+树作索引的存储结构?

A: 
  反例:
    二叉树: 节点只存一个数据,高度太高(一般用于内存查值)
    B树: 节点含数据,降低了每页中索引节点的数量
  原因:
    - IO代价更低,索引节点不含数据,一个盘块包含更多的索引节点
    - 查询效率稳定,任何查询都是从根节点到叶子节点
    - 便于遍历,数据节点在最底层且通过链表连接
    - 便于范围查找,数据节点已排好序,位于最底层
